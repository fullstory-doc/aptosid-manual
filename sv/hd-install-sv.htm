Förberedelser för hårddiskinstallation

För normalt bruk rekommenderar vi ext4, det är standardfilsystemet för aptosid.

Innan installationen bör du avlägsna usb-minnen, kameror, etc.. Installation till USB-enheter kräver fler steg. Du kan editera installer-filen: ~/.sidconf, och därmed använda ett annat filsystem eller sprida din installation över olika partitioner. T.ex en separat /home.

Det är starkt rekommenderat att använda en separat datapartition. Fördelarna när det kommer till katastrofåterställning och stabilitet av din data är omätbara.

Därför kommer $HOME bli din plats där programkonfigurationer sparas. Eller för att beskriva det på ett annat sätt: En behållare för dina program att spara sina inställningar.

Ominstallering av applikationer för att återuppbygga eller duplicera till en annan dator

För att göra en lista av dina installerade applikationer så att du kan installera desamma på en annan dator, eller kanske för att du av någon anledning ominstallerar på din egen dator. I en terminal:

dpkg -l|awk '/^ii/{ print $2 }'|grep -v -e ^lib -e -dev -e $(uname -r) >/home/username/installed.txt

Sedan kopierar du över textfilen till ett usb-minne eller någon annan flyttbar disk.

På den nya datorn kopierar du textfilen till $HOME och använd listan som en referens för att installera önskade program.

RAM och Swap

På datorer med mindre än 512mb swap måste du ha en swap-partition. Storleken bör inte vara mindre än 128mb (cfdisk-utdata skall man ej lita på då den räknar med en 10-bas.), mer än 1gb swap är sällan skäligt, såvida du inte kopierar stora filer eller om du behöver använda suspend-to-disk/hibernate och serversystem. Om detta är fallet skapar du som ett minimum en 2gb-swap.

Vänligen se: Partitionera din hårddisk

GÖR ALLTID BACKUP På DIN DATA och även på bokmärken och e-post! See Back-Up with rdiff and Back-Up with rsync. Another option is sbackup (needs installing).

Installation till hårddisken är bekvämare och snabbare än att köra ett system från en live-CD.

Först behöver du ställa in "boot order" i BIOS så att CD-ROM står först. Med de flesta datorer kan du gå in i BIOS genom att trycka på del-knappen vid uppstart. Med vissa BIOS kan man välja "boot device" direkt genom att trycka fram en meny med t.ex F8.

aptosid bör i de flesta fall nu starta upp. Om så inte är fallet kan du använda dig av olika uppstartsalternativ (dessa kallas "cheatcodes") som kan anges i starthanteraren. Användandet av uppstartsparametrar (e.x. för skärmupplösning eller språkval) kan spara en massa tid med förinstallationskonfigurationen. Se även "Cheatcodes" and VGA-upplösningar

(U)EFI booting

The bootloader will be an EFI program installed to /boot/efi/EFI/debian/grubx64.efi within your EFI system partition and mounted below /boot/efi/ on your installed system, provided the following conditions are met:

  • The motherboard needs to have EFI firmware with EFI booting enabled.
  • A current aptosid-amd64.iso
  • Booted using UEFI and this is apparent from the plain white/blue grub2 menu, instead of the usual graphical boot menu provided by isolinux for BIOS booting, on the live medium.
  • You have a GPT disk with an EFI System Partition (type EF00). This must be vfat or FAT formatted.
  • The install target is not a USB disk. If you need to boot a USB stick see UEFI bootable removable devices.

UEFI bootable removable devices

In order to create a UEFI bootable device, such as a USB stick, the contents of the aptosid ISO can be extracted to a vfat formatted partition and the partition marked as bootable.

Optionally this removable medium can also be made compatible with PC-BIOS booting by installing a traditional boot sector in addition to the EFI boot loader (assuming /dev/sdz1 to be the target partition on the USB stick).

syslinux -i -d /boot/isolinux /dev/sdz1
install-mbr /dev/sdz

UEFI Secure Boot

aptosid does not currently support UEFI Secure Boot. The current intention is to wait to see how Debian will handle UEFI Secure Boot before deciding the best course of action for aptosid.

In order to allow dual/triple/quadruple booting of aptosid with other operating systems, UEFI Secure Boot needs to be disabled in the system firmware. The board firmware should offer a boot menu, although on some boards this may mean you need to boot into the MS OS Win 8 and cycle through various menus to turn off UEFI Secure Boot and then reboot.

Rod Smith of rodsbooks.com provides a good insight on the subject of Secure Boot: Managing EFI Boot Loaders for Linux: Dealing with Secure Boot.

Partitioning GPT and UEFI disks

For partitioning GPT and UEFI disks consult: Partitioning with gdisk for GPT disks.

Choosing the language for your installation

Language Installs with KDE-full

Select your main language from the grub menu (F4) in the kde-full release, to install the localisations for the desktop and many applications while booting.

This ensures they are also present after installing aptosid, while only installing the required languages for the given system. The amount of memory required for this feature depends on the language and aptosid may refuse to install the given language packs automatically with insufficient RAM and the boot sequence will be continued in English language but with the desired locales settings (currency, date and time format, keyboard charsets). 1 GB memory or more should be safe for all supported languages, which are:

Default - Deutsch (German)
Default - English (English-US)
*Čeština (Czech)
*Dansk (Danish)
*Español (Spanish)
*English (GB)
*Français (French)
*Italiano (Italian)
*Nihongo (Japanese)
*Nederlands (Dutch)
*Polski (Polish)
*Português (Portuguese BR and PT)
*Română (Romanian)
*Русский (Russian)

The language selection depends on the availability of aptosid-manual translations, get involved to add your language.

Other Language installs with KDE-lite
  1. Select your main language from the gfxboot menu (F4). (See also aptosid specific Live-CD Cheatcodes). The Language files themselves are not on the Live-CD so the system will fall back to default English. However, this will make the correct language configuration needed for your preferred language and therefore no need to make any changes into the system, aside from the installation of the missing language files.
  2. Start the installation.
  3. Install to HD and reboot.
  4. After HD install, install the language of your choice and applications via apt-get.
First Time boot up to the HD

After booting up for the first time you will discover that aptosid has forgotten its network configuration. The network can be comfortably set up from Kmenu > Internet > Ceni. For additional WIFI/WLAN roaming please read this.

aptosid installationsprogram

1. Installationsprogrammet startas från skrivbordsikonen, K-Meny> System>aptosid-installer, eller via konsollen med:

su
install-gui.bash

2. Efter att du läst (och förstått) varningstexten går vi vidare till att välja en partition.

Välj nu till vilken partition du vill installera och vi upprättar monteringspunkter. Partitions for which we do not establish mount points for are mounted by the installer (the swap partition will be automatically mounted, when the system starts).

NOTE: If you have your root partition ('/") formatted with your preferred file system, you can uncheck the "format with" checkbox, provided that the drop box reflects the file system of your choice.

All other partitions will be placed as a /media/ partition. Here you have to select the root partition ("/") for your aptosid installation. However, it is at this moment you can also choose to create a data partition. A 'left-click' will activate your choices for each partition.

Have you backed up your data?

If you haven't partitioned your hard drive yet, click on the Execute button in the Start Part.-manager panel, also have a look at Partitioning your HD using Gparted or, if you want to use the shell, read Partitioning your HD

Du kan även starta dem från konsole/terminalen

su-to-root -X -c gparted

or

su
cfdisk

3. Som boot-hanterare använder aptosid GRUB, installera därför Grub till MBR! Om du gör ett annat val här bör du veta vad du gör. Du behöver i så fall editera andra boot-hanterare om du vill behålla dem.

Grub känner igen andra installerade operativsystem som t.ex Windows och lägger till dem i boot-menyn.

Du kan även ändra tidszon i detta fönster.

4. Välj användarnamn och lösenord samt lösenord till root-kontot. (Kom ihåg dem!). Tänk på att inte använda allt för enkla lösenord. To add additional users, do so after installation via the terminal with adduser..

5. Välj ett namn på installationen (Hostname:) Du kan döpa den till vad som helst som innehåller siffror och bokstäver. Däremot kan ett värdnamn inte inledas med en siffra.

Efter det väljer du om du vill att ssh skall startas automatiskt.

6. Denna fråga är sista chansen för att gå igenom de förändringar du gjort. Läs noggrannt igenom och om det är i sin ordning klickar du på Save Config and Next..

Vid det här laget är det möjligt att editera konfigurationsfilen för att sedan starta installationen med den ändrade konfigurationen. Installationsprogrammet gör inga kontroller och får inte klicka på 'back' då förändringarna du manuellt lade till kommer att försvinna.

Förändring av (~/.sidconf) är för avancerade linux-användare som vill göra sina egna specialförändingar i konfigurationsfilen eller som har ett speciellt partitioneringskrav som skulle avvisas av installationsprogrammet.

Should the installer detect that you are trying to install over an old $HOME it will warn you and will not let you proceed with the installation until you choose another user name.

För att påbörja installationen klickar du på Begin Installation Installationsförloppet tar, beroende på system, mellan 5-15 minuter. På äldre datorer kan det ta upp till 60 minuter.

Om förloppsindikatorn låser sig en stund, avbryt inte, ge den bara lite tid.

Klart! Tag ur cd:n ur cd-spelaren och starta om.

Första uppstarten

Efter att du för första gången startat aptosid kommer du upptäcka att aptosid har glömt av nätverkskonfigurationen. Denna måste således konfigureras igen.

Den som tidigare hade sitt nätverk automatiskt inställt (DHCP) måste återaktivera det med:

ceni

Du hittar detta program i Kmenu>Internet>ceni. Se även: Internet och nätverk

To add an existing aptosid $home partition to new installation fstab needs to be altered, refer to Moving /home.

Do not use or share an existing $home from another distribution as the $home configuration files in a home directory will conflict if you share the same username between differing distributions.

För att lägga till användare

För att lägga till en ny användare med automatiska gruppermissioner skriver du som root: 

adduser <ny_användare>

Tryck bara på enter, den tar själv hand om inställningar. Du kommer att bli tillfrågad om att skriva in lösenord två gånger.

aptosid-specifika ikoner (som manual- och IRC-ikon) måste du lägga till manuellt för den nya användaren.

För att ta bort en användare

deluser <användare>

Read

man adduser
man deluser

kuser kan även den skapa nya användare. Där måste du dock själv manuellt ställa in gruppermissioner.

About su-to-root -X -c <application>

Flertalet kommandon kräver att du kör dem som root. För att möjliggöra detta skriver du:

su

Det vanliga kommandot för att bli root är 'su' men genom att använda su-to-root -X -c <application>.

Refer to About Xapps whilst in X as root

Some KDE applications require dbus-launch in front of the application:

dbus-launch <Application>

Ett exempel där man kör ett X11-program via su-to-root -X -c är när du behöver justera en konfigurationsfil som kräver root-rättigheter med hjälp av t.ex kate eller när du vill genomföra en partitionering med gparted. Du kan också editera root-filer genom att högerklicka på dem och välja 'edit-as-root', skriva in ditt root-lösenord, och kdesu kontaktas i bakgrunden.

Till skillnad från 'sudo' kan inte någon bara skriva sudo och skada ditt system.

VARNING: Medan du är inloggad som root kommer inte systemet hindra dig från att genomföra otrevliga saker som att ta bort viktiga filer etc. Du måste vara helt säker på vad du gör eftersom det är möjligt att orsaka stor skada i ditt system.

Du bör aldrig vara root i konsollen/terminalen för att köra standardprogram som du vanligtvis kör som vanlig användare. T.ex e-postprogram, office, webbläsare osv.
