aptosid-*.iso - Live Mode root lösenord

Komihåg att: När du kör kommandon med root rättigheter, bör du veta vad du gör! För att surfa på internet behöver man inga root rättigheter.

aptosid-*.iso standard lösenord

På aptosid LiveCD är inget lösenord inställt. Så om du vill köra program som kräver root rättigheter har du lite olika val:

Enkla metoden, skriv bara:

su
För att ställa in ett (temporärt) root-lösenord:

Öppna Terminalen:

aptosid@0[aptosid]$ sudo passwd
Enter new UNIX password:
Retype new UNIX password:
passwd: password updated successfully
aptosid@0[aptosid]$

Nu kan du använda det här lösenordet resten av tiden du använder aptosid-Live-session.

sudo är inte supportat vid hårddisk installationer, se sudo och sux on how to set an alias.

Köra ett program som root från terminal

Skriver du su-to-root -X -c <application> i terminalen så blir du root med Xapp rättigheter.

Öppna terminalen:

aptosid@0[aptosid]$ su-to-root -X -c gparted  (eller vad du vill köra...)

En del KDE applikationer kräver dbus-launch kommandot framför applikationsnamnet:

dbus-launch <Applikationen>

Ett annat alternativ som är lika för de flesta skrivbordshanterare är:

Alt+F2 su-to-root -X -c <Applikationen>

T ex:

Alt+F2 su-to-root -X -c gparted

För att ta bort root rättigheter i terminalen skriv:

exit

eller klicka längst upp i högra hörnet och stäng terminalfönstret

Om du skulle bli utlåst från en aptosid-*.iso behöver du göra som följer för att sätta ett nytt lösenord:

alt+ctrl+F1
sudo passwd

När lösenordet är aktiverat, gör följande:

alt+ctrl+F7

Installera program från Live-CD

apt-get update
apt-get install ditt-önskade-paket

Kom ihåg: när du stänger av efter att ha använt en Live-CD, så sparas inga ändringar du gjort, om du inte har aktiverat fromiso och persist.
