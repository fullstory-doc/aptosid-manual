Partíciók méretezése példákkal

Normál használatra az ext4 fájlrendszert javasoljuk.

A GParted partíció-kezelővel partícionálhatjuk és/vagy formázhatjuk a merevlemezeket. A program grafikus felülettel rendelkezik és használata egyértelmű.

A Gparted képes partíciók mozgatására és azok méretének csökkentésére, illetve NTFS partíciók kezelésére is (azzal a megkötéssel, hogy egy NTFS partíció bármilyen változtatása után azonnal újra kell indítani a számítógépet a további műveletek elvégzése előtt). Itt megtalálható a gparted teljes dokumentációja. Az NTFS-partíciókon jogdíjas programokkal, mint pl. a Partition Magic (TM) és Acronis (TM) is végrehajthatjuk a szükséges változtatásokat.

MINDIG MENTSÜK LE AZ ADATAINKAT!

Ha a partíció fel van csatolva, akkor azt le kell csatolni (még akkor is, ha az cserehely (swap) partíció) a partíció ikonkára való jobb egérkattintással, vagy parancssorból:

umount /dev/sda1

A cserehely partíció lecsatolása terminálból: 

swapoff -a

Általános szabály, hogy a merevlemezes telepítésre 5Gb elegendő, de a mindennapi használathoz ez kevés lesz - a minimum ajánlott partíció méret 12 Gb. A linux-szal ismerkedők számára kezdésként két partíció (root/home és cserehely) létrehozását ajánljuk. Ez a felosztás egyszerűsíti a kezdeti telepítést, és később kiegészíthető /home és egyéb extra adat partíciókkal.

A cserehely partíció létrehozása mindenképp szükséges (ez a Windows swap fájljához hasonló, de annál hatékonyabb). Normál használat esetén a cserehely mérete a memória méretének kétszerese legyen.

Windows rendszerekkel történő adatcseréhez vfat (fat32) vagy ext2 típusú partíció használható (az ext2 partícióhoz létezik MS Windows™ eszközmeghajtó). [Az XFS nem támogatott]. Ext2 Installable File System For MS Windows and also Writing on NTFS partitions with ntfs-3g.

It is wise to write down the names of the partitions for reference.

Here are some simple examples for different partition sizes:
1 TB for dual boot MS Windows and Linux

sda1 50 GB NTFS MS Windows System
sdb1 100 GB ext4 / (includes home)
sda3 300 GB FAT32/ext2 Data for MS Windows System and Linux
sda2 550 GB ext4 Data for Linux
sda4 2 GB Linux Swap Linux Swap

sda1 30 GB NTFS MS Windows System
sda2 20 GB ext3 /
sda3 20 GB ext3 /home
sdb1 48 GB FAT32/ext2 Data exchange MS Windows and Linux
sda4 2 GB Linux Swap Linux Swap
60 GB for dual boot MS Windows and Linux:

sda1 24 GB NTFS MS Windows System
sda2 10 GB FAT32/ext2 Data for MS Windows and Linux
sda3 10 GB FAT32/ext2 Data for MS Windows and Linux
sdb1 14 GB ext4 / (includes home)
sda4 2 GB Linux Swap Linux Swap

General

There are many ways to partition your hard drives. These examples should be enough for a start.

It makes good sense to purchase a USB hard drive to make a regular data back up should any of your hard drives fail. If dual booting with MS Windows™, always put MS on the first hard disk/partition.

For other partitioning options see Partitioning to use LVM - Logical Volume Manager and Installing to encrypted partitions - cryptroot.
