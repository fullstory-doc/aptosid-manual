A aptosid-CD letöltése

Kizárólag jó minőségű CD (illetve DVD+RW) lemez használata, DAO (disk-at-once) mód kiválasztása és legfeljebb 8x-os írási sebesség szükséges a megfelelő minőség eléréséhez.

A tükörszervereken lévő fájlok leírása

Minden aptosid tükrön megtalálhatók az alábbi fájlok:

MD5SUM
MD5SUM.gpg
SHA256SUM
SHA256SUM.gpg
SOURCES
aptosid-20xx-xx-release-name-window-manager-arch-datetimestamp.arch.manifest
aptosid-20xx-xx-release-name-window-manager-arch-datetimestamp.iso

A .manifest fájl felsorolja az összes csomag nevét, amelyet az ISO tartalmaz.

A .iso a képfájl, amely letöltendő.

Az .md5 és .sha256 az ISO fájl integritás- (sérülés)ellenőrzéséhez szükségesek.

Az .gpg aláírás fájlok segítségével ellenőrizhető, hogy az integritás-fájlokat (.md5 és .sha256) nem változtatta meg rosszindulatú személy, tehát alkalmasak az ISO fájl integritásának ellenőrzésére.

The sources tar is only for those who require a static copy of the source code for redistribution purposes to comply with the licensing requirements. Read the SOURCES file for more information.

md5sum és ellenőrzés

FONTOS MEGJEGYZÉS: A aptosid, mint Linux LIVE-CD, erős tömörítési technológián alapul, ezért az ISO lemezképfájl lemezre írása odafigyelést igényel. Kizárólag jó minőségű CD (illetve DVD+RW) lemez használata, DAO (disk-at-once) mód kiválasztása és legfeljebb 8x-os írási sebesség szükséges a megfelelő minőség eléréséhez.

Az md5sum egy ellenőrző összeg, melyet állományok épségének ellenőrzésére használunk. Az ellenőrzés során az aktuális md5sum értéket egy ismert értékhez hasonlítjuk. E módon leellenőrizhető, hogy egy fájl megváltozott, vagy megsérült-e. Minden esetben ajánlott elvégezni az internetről letöltött fájlok ellenőrzését.

Ha a letöltött fájl md5sum értéke megegyezik az md5-fájlban olvasható értékkel, akkor biztos lehetsz benne, hogy a letöltés hibamentes. Linux alatt a következőképpen állíthatod elő az ellenőrző összeget: (a művelet némi időt igényel)

$ md5sum ellenorizni_kivant_fajl_neve

Az eredmény a konzolon fog megjelenni.
Az md5sum program (486 kb) Windows változatban is elérhető.

A aptosid ISO állományai mellé mindig biztosítja az md5 fájlt is, és CD-re írás előtt mindig ajánlatos azokat leellenőrizni. Ez garantálja, hogy bármilyen probléma esetén kizárható, hogy az a hibásan letöltött fájl miatt keletkezett, ezáltal a fórumot is tehermentesíthetjük az olyan hibajelentésektől, melyek egyértelműen visszavezethetők a hibásan letöltött fájlokra. Az ellenőrzés parancssori ablakban végezhető el. Lépj be abba a könyvtárba, ahol a letöltött ISO és az md5 fájl található, majd add ki a következő parancsot:

Download the following files from the mirror:

MD5SUM
MD5SUM.gpg
SHA256SUM
SHA256SUM.gpg

In a terminal:

$ md5sum aptosid.iso

Ha eltérés mutatkozik, hibaüzenetet kapsz:

"aptosid.iso: Error
md5sum: Warning: calculated checksum does not match!"

Ha letöltött fájl hibátlan, a program nem ír ki semmilyen üzenetet.

Az aláírás-fájlok (.gpg) ellenőrzéséhez felhasználóként a következőt kell tenni:

Example using:
aptosid-2011-02-imera-kde-lite-i386-201107131633.iso

1) "Stefan Lippers-Hollmann's" (slh) publikus kulcsának importálása a kulcsszerverről. (An alternate keyserver is subkeys.pgp.net ):

$ gpg --keyserver wwwkeys.pgp.net --recv-keys E3BD538B
gpg --edit-key E3BD538B
Command> trust
Command> (choose the option you prefer or quit)
Command> quit

2) Most már ellenőrizhetőek az integritás-ellenőrző (checksum) fájlok:

$ gpg --verify SHA256SUM.gpg SHA256SUM

Ha a kulcsot 'megbízható'-ként importáltad ("5 trusted ultimately") a szükséges gpg parancsok segítségével, a következő kimenetet kell látnod:

$gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID E3BD538B
$ gpg: Good signature from "Stefan Lippers-Hollmann (aptosid.com) <s.l-h@gmx.de>"

Amennyiben a kulcs nem megbízhatóként lett importálva, figyelmeztetést kapunk, bár az aláírás megfelelő ('Good signature'):

gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID E3BD538B
gpg: checking the trustdb
gpg: 3 marginal(s) needed, 1 complete(s) needed, PGP trust model
gpg: depth: 0  valid:   2  signed:   0  trust: 0-, 0q, 0n, 0m, 0f, 2u
gpg: Good signature from "Stefan Lippers-Hollmann (aptosid.com) <s.l-h@gmx.de>"
gpg: WARNING: This key is not certified with a trusted signature!
gpg:          There is no indication that the signature belongs to the owner.
Primary key fingerprint: 64C3 6120 DA8D 91E7 378B  E79F 3916 C431 F809 94F6

A CD képfájl kiírása Windows alatt

FONTOS MEGJEGYZÉS: A aptosid, mint Linux LIVE-CD, erős tömörítési technológián alapul, ezért az ISO lemezképfájl lemezre írása odafigyelést igényel. Kizárólag jó minőségű CD (illetve DVD+RW) lemez használata, DAO (disk-at-once) mód kiválasztása és legfeljebb 8x-os írási sebesség szükséges a megfelelő minőség eléréséhez.

Természetesen a CD Windows alatt is megírható. A letöltött fájlt ISO-lemezképként kell kiírni. Zavaró lehet, ha a Winrar (vagy valamely más tömörítő alkalmazás) hozzá lett rendelve a .ISO fájlokhoz, mert a rendszer tömörített állományként kezelheti. Az ISO-fájlból CD-t kell készíteni.

Többféle jó megoldás létezik Windows alatt.

CD írás Linux alatt

FONTOS MEGJEGYZÉS: A aptosid, mint Linux LIVE-CD, erős tömörítési technológián alapul, ezért az ISO lemezképfájl lemezre írása odafigyelést igényel. Kizárólag jó minőségű CD (illetve DVD+RW) lemez használata, DAO (disk-at-once) mód kiválasztása és legfeljebb 8x-os írási sebesség szükséges a megfelelő minőség eléréséhez.

Ha már Linux fut a gépen, akkor a CD-t meg lehet írni parancssori eszközzel vagy más, erre a célra telepített programmal.

A aptosid és más Linux disztribúciók alapértelmezésként szállítják a K3b cd-író programot. Az elindítás után a menüből kell kiválasztani az "Eszközök" -> "Írás CD képmásból...", illetve angol nyelvű rendszeren "Tools" -> "burn CD-Image...", majd ki kell választani a letöltött ISO-fájlt. (pl. aptosid.iso).

A K3B először kiszámolja az ISO fájl MD5-sum ellenőrző összegét (ez eltarthat egy ideig). Ha az ellenőrző összeg megegyezik az MD5-fájlban található értékkel (pl. aptosid.iso.md5) akkor a letöltés sikeres volt és az írás megkezdhető az "Indítás" gomb megnyomásával.

Néha előfordulhatnak problémák a grafikus felületű programok használata során. A fájl kiírható parancssorból is a burniso segítségével.
