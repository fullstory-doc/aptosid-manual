aptosid LAMP csomag

A LAMP rövidítés szabad szoftverek egy csoportjára utal, amelyeket gyakran használnak együtt dinamikus web-szolgáltatások vagy szerverek létrehozásához.
* Linux, az operációs rendszer
* Apache, a webszerver
* MySQL, az adatbázis-kezelőre rendszer (más néven adatbázis-szerver)
* Perl, PHP, és/vagy Python, szkript nyelvek

FIGYELMEZTETÉS: Soha ne használd a napi használatú PC-det internet webszerver célokra! Erre dedikált szervert érdemes használni, ami semmi egyebet nem csinál!

Egy szerver felhasználási módjai:
a) Helyi teszt szerver webdizájnerek részére, internet kapcsolat nélkül - a továbbiakban ezzel fogunk foglalkozni,
b) egy privát "homokozó" jellegű szerver, amely az internetre csatlakozik,
c) privát webszerver, amelyet kiajánlunk az internet irányába,
d) üzleti webszerver, amely meghaladja ennek a kézikönyvnek a határait.

Minimális hardver követelmények

Legkevesebb 256Mb memória. Ennél kevesebb memória használata során biztosan problémák fognak jelentkezni, mert a MySQL adatbázis-kezelő a megfelelő működéshez sok memóriát igényel. A Mysql "cannot connect to mysql.sock" hibákat fog üzenni kevés memória esetén.

The packages you will need to install are:

apache2
apache2-utils
apache2-mpm-prefork
php5 php5-common
mysql-server
mysql-common
libapache2-mod-php5
php5-mysql
phpmyadmin

FIGYELEM

apt-get remove --purge splashy

A splashy csomag akadályozza a mysql megfelelő működését.

Az Apache beállító fájljának elérési helye: /etc/apache2/apache2.conf. A web gyökérkönyvtár pedig a /var/www alatt érhető el.

A PHP telepítésének és megfelelő futásának ellenőrzéséhez készítsünk egy test.php fájlt a /var/www mappa alá a phpinfo() funkcióval, pontosan ahogy az alábbiakban látszik:

mcedit /var/www/test.php

# test.php
<?  phpinfo(); ?>

Írjuk be a böngészőbe:

http://localhost/test.php
vagy
http://sajátipcím:80/test.php

Ekkor meg kell jelenjenek a PHP beállítások és használható funkciók.

Az Apache konfigurációs fájlban állíthatók be a szükséges módosítások, esetleg a virtuális domain-ek.

A telepítés ellenőrzéséhez írjuk be a böngésző címsorába:

http://sajátipcím/apache2-default/

Hibátlan telepítés esetén az Apache alapértelmezett képernyőjét kell látnunk.

Az apache2 esetén az alapértelmezett document root könyvtár a /var/www. Változtassuk meg az alábbi módon: 

mkdir /home/myself/www
ln -s /home/myself/www /var/www

A fentiek elvégzése után a saját website-ot szerkeszthetjük saját HOME könyvtárunkban, felhasználói jogainkkal.

A szerver fájlok védelme alapértelmezetten

Az Apache gyakran félreértett funkciója az alapértelmezett hozzáférés (default access). Ennek lényege, hogy - amíg nem változtatunk rajta - ha egy fájl elérhető a megfelelő URL segítségével, azt a kliens rendelkezésére bocsátja.

Vegyük a következő példát:

 1. # cd /; ln -s / public_html
 2. Írjuk be a böngészőbe http://localhost/~root/

Ez a beállítás a teljes fájlrendszer böngészését engedélyezi a kliensek számára! Ennek elkerülésére adjuk hozzá a következőt a webszerver konfigurációjához:

<Directory />
   Order Deny,Allow
   Deny from all
</Directory>

Ez megtiltja az alapértelmezett hozzáférést a fájlrendszerhez. A <Directory> blokkok segítségével ki kell alakítanunk olyan szabályokat, hogy csak a megfelelő tartalomhoz lehessen kívülről hozzáférni. Például,

<Directory /usr/users/*/public_html>
    Order Deny,Allow
    Allow from all
</Directory>
<Directory /usr/local/httpd>
    Order Deny,Allow
    Allow from all
 </Directory>

Különösképpen figyeljünk a <Location> és a <Directory> direktívák együttes hatására; tehát még ha a <Directory /> meg is akadályozza a hozzáférést, a <Location /> direktíva megfordíthatja az egészet.

Szintén kezeljük körültekintően a UserDir direktívát; ha beállítunk olyamit, mint pl. "./", akkor az első példában szereplő hatást érjük el vele. Apache 1.3 vagy újabb használatának esetén kiemelten ajánljuk a következő sor használatát a konfigurációban:

UserDir disabled root

# apache2-ssl-certificate

Futtassuk az “a2enmod ssl” szkriptet:

# a2enmod ssl

Ezzel automatikusan generálunk egy szimbolikus linket a mods-available és a mods–enabled könyvtárak között.

Készítsünk másolatot a '/etc/apache2/sites-available/default' fájlról a /etc/apache2/sites-available/ alá - nevezzük el 'ssl'-nek.

# cp /etc/apache2/sites-available/default /etc/apache2/sites-available/ssl

Készítsünk symlinket az új konfigurációhoz:

#  ln -s /etc/apache2/sites-available/ssl /etc/apache2/sites-enabled/
(vagy)
#a2ensite ssl

Az alap konfigurációs beállításokat a /etc/apache2/apache2.conf alatt tudjuk elvégezni, illetve megváltoztathatjuk az alapértelmezett document root könyvtárat a /etc/apache2/sites-available/default fájlban, ezután indítsuk újra az Apache szervert.

Az Apache szerver újraindításához az alábbi parancs használható:

#service apache2 restart

Ezek után szükségünk van az alapértelmezett port megváltoztatására a /etc/apache2/ports.conf-ban, mivel az Apache alapértelmezésben a 80-as porton figyel; mivel mi az SSL-t állítjuk be, át kell ezt állítanunk a 443-as portra.

 Listen 443

Szerkesszük a /etc/apache2/sites-available/ssl (vagy amilyen néven elneveztük az új ssl konfigurációnkat) fájlt és változtassuk meg a 80-as portot a site nevénél 443-ra.

A /etc/apache2/apache2.conf fájlba írjuk be bárhová:

  SSLEngine On
  SSLCertificateFile /etc/apache2/ssl/apache.pem

A "SSLCertificateFile /etc/apache2/ssl/apache.pem" sort szerkesszük úgy, hogy az elérési út az általunk generált tanúsítványra mutasson. Egy példa:

SSLCertificateFile /etc/apache2/ssl/online.test.net.crt
SSLCertificateKeyFile /etc/apache2/ssl/online.test.net.key

Állítsuk a ServerSignature változót "off"-ra; a /etc/apache2/apache2.conf fájlhoz adjuk hozzá a következő két sort:

  ServerSignature Off
 ServerTokens ProductOnly

Ha engedélyezni szeretnéd a különféle alapértelmezett index fájl típusokat, nézd át az alábbi sort az /etc/apache2/apache2.conf fájlban

 DirectoryIndex index.html index.cgi index.pl index.php index.xhtml index.shtml

Indítsd újra az Apache szervert

 service apache2 restart

A "homokozó" teszt szerver elkészült, ha szeretnéd az internetre csatlakoztatni, NE TEDD...! Használj másik PC-t, amely pusztán arra a feladatra van dedikálva, hogy internet web szerver legyen!
