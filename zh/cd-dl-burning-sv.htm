Definitioner av filer på varje spegel

Varje aptosid spegel har följande filer:

MD5SUM
MD5SUM.gpg
SHA256SUM
SHA256SUM.gpg
SOURCES
aptosid-20xx-xx-release-name-window-manager-arch-datetimestamp.arch.manifest
aptosid-20xx-xx-release-name-window-manager-arch-datetimestamp.iso

The .manifest filen listar alla paket som ingår i den specifika iso filen.

The .iso är avbilden man ska ladda ner.

The .md5 och .sha256 filerna är till för att kolla integriteten på iso filen.

The .gpg filerna är signatur filer, så du kan verifiera att checksum (.md5 .sha256) inte har blivit ändrade och att dom kan användas för att kontrollera integriteten på iso filen.

sources tar är bara för de som kräver en statisk kopia av källkoden för vidaredistributions syften .Läs i SOURCES filen för mer information.

md5sum och validering

DETTA ÄR VÄLDIGT VIKTIGT: aptosid, som en Linux LIVE-CD, är baserad på en högkompressionsteknologi, och på grund av det, krävs följande då du ska bränna ISO-avbilden: Använd endast skivor av hög kvalitet och bränn i DAO-läge (disk-at-once) och inte snabbare än x8.

Detta är en kontrollsumma som används för att kontrollera integriteten hos en fil. Den nuvarande md5summan kontrolleras mot en känd kontrollsumma. På detta sätt kan du validera filen och se om filen blivit skadad eller förändrad. Detta är ett rekommenderat förfarande för filer som laddats ned från internet.

Om md5summan överensstämmer med den i filen kan du vara säker på att filen laddats ned korrekt. I Linux kontrollerar du summan med: (det tar en liten stund att beräkna denna)

$ md5sum fil_att_kontrollera

Summan skrivs ut till konsollen.
Med md5sum (486 kb) kan md5sum även bli validerad i Windows.

Till aptosid ISO-avbilder finns alltid en md5sum-fil och denna skall alltid kontrolleras innan du bränner ut avbilden. Detta garanterar att om fel uppstår, så återfinns de inte i den nedladdade filen, och detta gör att forumen hålls fria från problem som inte kan härledas eftersom filen var felaktig från början. Du bör göra detta i en konsoll. Gå till katalogen med ISO-filen och md5sum-filen och kör kommandot för att kontrollera att summan stämmer. T.ex:

Ladda ner följande filer från en nedladdningsspegel:

MD5SUM
MD5SUM.gpg
SHA256SUM
SHA256SUM.gpg

I Terminalen:

$ md5sum aptosid.iso

Skulle de inte stämma får du ett felmeddelande:

"aptosid.iso: Error
md5sum: Warning: calculated checksum does not match!"

Om filen är OK avslutas programmet utan att ge något meddelande.

Om du skulle vilja kontrollera äktheten på .gpg gör som följer som $USER:
Exempel fil:
aptosid-2011-02-imera-kde-lite-i386-201107131633.iso

1) Import "Stefan Lippers-Hollmann's" (slh's) public key från en key-server. (An alternate keyserver is subkeys.pgp.net ):

$ gpg --keyserver wwwkeys.pgp.net --recv-keys E3BD538B
gpg --edit-key E3BD538B
Command> trust
Command> (gör ditt val eller quit)
Command> quit

2) Nu kan du verifiera integriteten på checksum med:

$ gpg --verify SHA256SUM.gpg SHA256SUM

Om du har klassificerats nyckeln som pålitlig med hjälp av nödvändig gpg kommandon, så är detta det du ska få med "5 trusted ultimately":

$gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID E3BD538B
$ gpg: Good signature from "Stefan Lippers-Hollmann (aptosid.com) <s.l-h@gmx.de>"

Du kommer få 'Good signature' med en WARNING om du valde att inte klassificera nyckeln:

gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID E3BD538B
gpg: checking the trustdb
gpg: 3 marginal(s) needed, 1 complete(s) needed, PGP trust model
gpg: depth: 0  valid:   2  signed:   0  trust: 0-, 0q, 0n, 0m, 0f, 2u
gpg: Good signature from "Stefan Lippers-Hollmann (aptosid.com) <s.l-h@gmx.de>"
gpg: WARNING: This key is not certified with a trusted signature!
gpg:          There is no indication that the signature belongs to the owner.
Primary key fingerprint: 64C3 6120 DA8D 91E7 378B  E79F 3916 C431 F809 94F6

Hur bränner jag i Windows?

DETTA ÄR VÄLDIGT VIKTIGT: aptosid, som en Linux LIVE-CD, är baserad på en högkompressionsteknologi, och på grund av det, krävs följande då du ska bränna ISO-avbilden: Använd endast skivor av hög kvalitet och bränn i DAO-läge (disk-at-once) och inte snabbare än x8.

Självklart kan du även bränna ut avbilden i Windows. Den nedladdade filen måste brännas som en ISO-fil. Om Winrar (eller något annat arkiveringsverktyg) är inställt för att hantera ISO-filer kan filen misstas för att vara ett arkiv. Du måste bränna ut ISO-filen som en avbild.

Det finns flera bra val för att bränna i Windows.

Open Source brännarprogram för Windows

cdrtfe: Kompatibel med Windows 9x/ME/2000/XP (testad med Win95, Win98SE, Win2000, and WinXP) För Win9x/ME krävs ASPI Layer (ex. Adaptec ASPI 4.60)

LinuxLive USB Creator, ett open source projekt, som erbjuder ett GUI för MS Windows™, som gör det möjligt att installera aptosid-i386.iso (32 bit) till ett USB minne. Du behöver använda fromhd bootline/cheatcode vid kommandoprompten.

Closed Source och proprietära brännarprogram för Windows
  • Nero (inte nero express!)
  • Ett annat bra freeware-verktyg är CD/DVD Burner XP pro
  • BurnCDCC från terabyteunlimited kan endast bränna ISO-filer.

Hur bränner jag i Linux?

DETTA ÄR VÄLDIGT VIKTIGT: aptosid, som en Linux LIVE-CD, är baserad på en högkompressionsteknologi, och på grund av det, krävs följande då du ska bränna ISO-avbilden: Använd endast skivor av hög kvalitet och bränn i DAO-läge (disk-at-once) och inte snabbare än x8.

Om du redan har Linux installerat på din dator kan du bränna en skiva från terminalen eller med hjälp av annat installerat brännarprogram.

Med aptosid och andra Linux-distributioner kommer K3b förinstallerat. öppna det, välj "Tools" -> "Bränn CD-avbild..." och välj sedan ISO-filen du vill bränna ut. (t.ex aptosid.iso)

K3B kontrollerar md5summan hos ISO-filen (detta tar en stund). När kontrollsumman stämmer överens med den i md5sum-filen var nedladdningen av filen lyckad och du kan bränna ut filen genom att klicka på "Start".

Problem med bränning uppstår oftast när man använder ett GUI. Du kan bränna direkt från kommandoprompten med burniso.

Se också Bränna aptosid till ett USB/SSD minne med Linux, MS Windows eller Mac OS X system (32 och 64 bit).
