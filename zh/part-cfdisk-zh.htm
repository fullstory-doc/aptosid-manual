磁盘及分区命名

警告：关于磁盘及分区命名请参考UUID，分区标记几fstab，因为sidux现在默认使用UUID

当前命名实例

对磁盘

自udev采用通用唯一标志码（Universal Unique Identifier）UUID起，伴随新近linux内核的出现，所有的块设备都用一种三字母表示外加数字的命名方案，基本是用sda表示磁盘设备而sdaX表示硬盘分区。

不管他们用什么标准，PATA（IDE），SATA（串行ATA）或SCSI，从现在起区分你机器里的不同磁盘的方法只有一种/dev/sda1，/dev/sdb1，/dev/sdc1，/dev/sdd1等。

当你用鼠标运行sidux livd-cd及HD安装之后的桌面上的媒介图标时，你就会看到弹出的窗口是以这种方式显示你的设备的。

我们大力鼓励你自己自己造一张表，不管是用手还是计算机工具，将你捕捉到的你的计算机上的所有块设备的细节通通罗列在表里面。枯燥是当然的，但这一行动可能在将来会为你节省很多时间同时也省却你很多麻烦。

live-cd上或HD安装之后的sidux/etc/fstab文件将/dev/sdaX信息保留在了方括弧内，嵌在了每一设备所在行上方的注释行里。例如（黑体标注的仅作示例）。

# added by sidux [/dev/sdd1, no label]
UUID=2ae950df-7d72-4d9b-a71a-bad1eb2d4f6a       /       ext4    defaults,errors=remount-ro,relatime      0 1
对分区

如上所见，对分区/dev/磁盘标志符还辅以一数字。

有如下分区类型：预选，拓展和逻辑，其中逻辑又含于拓展。最多有4个预选或3个预选加1个拓展。拓展可包含至多11个逻辑分区。

预选或拓展的名字在sda1到sda4之间取。逻辑分区往往紧接着往后，且为一拓展分区的一部分。你可以（通过libata）定义最多11个这样的分区和它们的表示，从数字5开始（比如sda5），直到15结束（sda15）。

一些应用的例子

/dev/sda5：只可能是一个逻辑分区（在本例里，是它所在磁盘设备的第一个分区），很可能位于你计算机的第一个SATA或第一个IDE磁盘上（视你的BIOS如何设置而定）。

/dev/sdb3：只可能是一个预选分区或一个拓展分区；磁盘字母不同于第一个例子里的，我们只可以说这一分区绝不可能位于同一个设备上。

早前而现已作废的对IDE设备的表示

在旧一点linux系统上，IDE磁盘设备（PATA）用hdaX 而非sdaX来命名以同那些使用当前标准的区分开。

用cfdisk对你的HD进行分区

一般使用我们推荐ext4文件系统，它是sidux默认的文件系统，并且维护得很好。

打开一个konsole／xterm，变身root用户然后开始cfdisk： （如果你在完成HD安装后的系统上，你得敲入你的root密码）

su
cfdisk /dev/sda
用户界面

在第一屏上cfdisk列出了当前的分区表，每一分区的名字和与一些相关数据都有列出。在屏幕下方，有一些命令按钮。要在分区间进行切换，用移动光标的上下箭头键。要在命令间切换，用移动光标的左右箭头键

删除一个分区

要删除一个分区，用上下键以高光把它标志出来，再用左右键选择

Delete

命令，然后按

Enter
创建一个新的分区

要创建一个新的分区，用命令：

New

（用左右键选择它），然后按回车。你必须决定它为一个预选或逻辑分区。如果你想要一个逻辑分区，该程序会自动为你创建一个拓展分区。然后你必须选择分区大小（以MB为单位）。如果你不能以MB为单位键入值，用Esc键回到主屏，再用命令

Units

选择MB

分区类型

要为Linux swap或Linux设定分区类型，高亮标志出当前的分区，再用命令：

Type
.

你会得到一张不同类定的列表。按空格键，你会得到更多。找出你需要的类型，然后在提示那里敲入对应的数字。（Linux swap类型为82，Linux文件系统应该取类型83）

使一个分区可引导

对Linux无需预设一个可引导的分区，但其他一些OS需要。高亮标志出该分区，然后选择命令（注意：当安装在一个外接HD上时，必须有一个分区可引导）：

Bootable
把结果写至磁盘

当一切都做完之后你可以用Write命令写下你的变更。分区表会写往磁盘（如果你得到一个涉及到dos的错误，你可以忽略它）。由于这会破坏你已删除或变更的分区上的所有数据，在你真正按这个键之前你应该十分确定你确实要这么做。

Return
Quit

要退出该程序，选择Quit命令。在离开cfdisk之后，开始格式化或安装之前，你应该重启你的匣子，这样sidux得以重读新的分区表。

格式化分区（用cfdisk分区之后）

基础

一个分区必须有一个文件系统。Linux知道使用不同的文件系统。有ReiserFS，Ext4，以及面向有经验的用户的XFS和JFS。Ext2作一存储格式很好，因为一个供数据交换的Windows驱动器是现成的。 Ext2 Installable File System For MS Windows.

一般使用我们推荐ext4文件系统，它是sidux默认的文件系统，并且维护得很好。

格式化

关掉cfdisk之后我们回到控制台。进行格式化你需要成为root用户。格式化root与／或home分区，本例中为sdb1，我们敲入：（如果你在完成HD安装后的系统上，这儿你得敲入你的root密码）

su
mkfs -t ext4 /dev/sdb1

会出现一个问题，如果你确定你已经选对了分区，回答是（yes）。

当命令完成之后，你会得到提示，告诉你ext4格式化成功写往磁盘。如果你没有得到这个提示，在用cfdisk分区时可能出了什么问题，或者sdb1不是一个linux分区。这种情况下你可以用下面的命令进行检测

fdisk -l /dev/sdb

如果有什么不对，可以再重新分区。

如果格式化成功，你想要一个单独的home分区的话，对它执行同样的过程。

最后我们格式化swap分区，本例中为hdb3

mkswap /dev/sdb3

那之后：

swapon /dev/sdb3

然后我们在控制台敲入：

swapon -s

检测swap是否被识别。新挂载的swap现在应该被识别了，我们的情形为：

文件名 类型 大小 已用 优先级
/dev/sdb3 partition 995988 248632 -1

如果swap被正确识别，我们敲入

swapoff -a

然后重启。

现在一切就绪，你可以开始安装了。
