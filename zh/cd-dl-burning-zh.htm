Definitions of files on each mirror

Each sidux mirror has following files:

MD5SUM
MD5SUM.gpg
SHA256SUM
SHA256SUM.gpg
SOURCES
sidux-20xx-xx-release-name-window-manager-arch-datetimestamp.arch.manifest
sidux-20xx-xx-release-name-window-manager-arch-datetimestamp.iso

The .manifest file lists all the packages contained in the specific iso.

The .iso is the image to download.

The .md5 and .sha256 files are to check the integrity of the iso.

The .gpg files are signature files, so you can verify that the checksums (.md5 .sha256) have not been changed and that they can be used to verify the integrity of the iso.

The sources tar is only for those who require a static copy of the source code for redistribution purposes to comply with the licensing requirements. Read the SOURCES file for more information.

Thank you to webtropia.com who sponsor aptosid and to all other mirror hosters for their efforts regarding aptosid.

If anybody has a fast FTP-Server with sufficient traffic at their command, please inform the sidux team on the sidux Forums or on our IRC-Channel on oftc.net.

md5和与验证

这点至关重要： sidux，系一Linux LIVE-CD，基于高容压缩技术，正因为如此，刻录ISO映像文件时要特别注意。只使用高质的CD－媒介［或DVD＋RW］，在DAO－模式（disk-at-once，磁盘同步）下刻录且速度不超过x8。

这是一个文件的校验和，用来检查文件的完整性。通过将文件当前的md5和与一个已知的校验和进行比对，你可以验证文件是否被修改或损坏；对从因特网上下载的文件建议这么做。

如果下载的文件的md5和与md5文件里的一致，你可以确定文件下载无误。在Linux下，你通过如下的命令获取校验和（计算会花上一点时间）

$ md5sum 要检查的文件

算出的和会写往konsole。
通过md5sum（486 kb），md5和也可以在Windows里进行验证。

sidux的ISO映像总是与一个md5和文件一同提供，在刻录之前务必先进行验证。这保证了如果出现问题，不会往下载的文件里去找，进而使论坛避免了那类根本就无法解释的问题，因为下载的ISO文件一开始就是受损的。转换到包含ISO文件与MD5和的目录，发出命令检查md5和，比如：

Download the following files from the mirror:

MD5SUM
MD5SUM.gpg
SHA256SUM
SHA256SUM.gpg

In a terminal:

$ md5sum sidux.iso

如果他们不匹配，你会得到一条错误：

"sidux.iso: Error
md5sum: Warning: calculated checksum does not match!"

如果下载的文件正确无误，程序无消息地终止。

If you wish to be able to verify the validity of the .gpg files do the following as $USER:
Example using:
aptosid-2011-02-imera-kde-lite-i386-201107131633.iso

1) Import "Stefan Lippers-Hollmann's" (slh's) public key from a key-server. (An alternate keyserver is subkeys.pgp.net ):

$ gpg --keyserver wwwkeys.pgp.net --recv-keys E3BD538B
gpg --edit-key E3BD538B
Command> trust
Command> (choose the option you prefer or quit)
Command> quit

2) Now you can verify the integrity of the checksum with:

$ gpg --verify SHA256SUM.gpg SHA256SUM

If you have classified the key as trustworthy using the necessary gpg commands, then this is the output you should get with "5 trusted ultimately":

$gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID E3BD538B
$ gpg: Good signature from "Stefan Lippers-Hollmann (sidux.com) <s.l-h@gmx.de>"

You will get 'Good signature' with a WARNING if you did not choose to classify the key:

gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID E3BD538B
gpg: checking the trustdb
gpg: 3 marginal(s) needed, 1 complete(s) needed, PGP trust model
gpg: depth: 0  valid:   2  signed:   0  trust: 0-, 0q, 0n, 0m, 0f, 2u

我用的Windows怎么刻录CD呢？

这点至关重要： sidux，系一Linux LIVE-CD，基于高容压缩技术，正因为如此，刻录ISO映像文件时要特别注意。只使用高质的CD－媒介［或DVD＋RW］，在DAO－模式（disk-at-once，磁盘同步）下刻录且速度不超过x8。

你用的Windows，自然也可以刻录CD。下载的文件必须作为ISO文件刻录。如果Winrar（或者任何其他的压缩工具）与.ISO文件链接了的话，可能比较烦人一点因其会把它当作一个压缩包。你必须把ISO文件刻录到一张CD上。

在Windows上刻录你有几个不错的选项。

面向Windows的开源刻录工具

cdrtfe：与Windows 9x／ME／2000／XP兼容（已与Win95，Win98SE，Win2000，及WinXP一起进行了测试）；Win9x/ME另需：工作的ASPI［译注：Advanced SCSI Programming Interface，高级SCSI编程接口］层（比如Adaptec ASPI 4.60）

LinuxLive USB Creator, an open source project, offers a GUI for MS Windows™, which enables an install of a sidux-i386.iso (32 bit) to a USB stick. You need to use fromhd bootline/cheatcode on the boot line.

面向Windows的闭源及私有刻录工具
  • Nero（不是nero nero专业版！）
  • 另一不错的免费软件工具是CD/DVD Burner XP pro
  • 来自terabyteunlimited的BurnCDCC只能刻录ISO。

用Linux刻录CD

这点至关重要： sidux，系一Linux LIVE-CD，基于高容压缩技术，正因为如此，刻录ISO映像文件时要特别注意。只使用高质的CD－媒介［或DVD＋RW］，在DAO－模式（disk-at-once，磁盘同步）下刻录且速度不超过x8。

如果在你的机器上已经运行着Linux，你可以利用shell或任何你已安装的刻录程序来刻录CD。

sidux及其他Linux发行版［译注：以KDE为桌面环境的，GNOME桌面环境一般为Bresero］，K3b为默认的cd刻录工具。打开它，选择“额外” -> “刻录CD映像...”然后选择ISO文件进行刻录（比如sidux.iso）。

K3B先计算ISO文件的MD5和（这会花上一点时间）。如果校验和与MD5文件（比如sidux.iso.md5）里的校验和相同，你下载成功，可以点击“开始”刻录文件。

Problems with burning mostly happen with a frontend. You can burn directly from the shell with burniso.

Also see Writing sidux to a USB/SSD stick with any Linux, MS Windows or Mac OS X system (32 and 64 bit).
