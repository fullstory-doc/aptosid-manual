sidux快速上手指南

这点至关重要： sidux，系一Linux LIVE-CD，基于高容压缩技术，正因为如此，刻录ISO映像文件时要特别注意。只使用高质的CD－媒介［或DVD＋RW］，在DAO－模式（disk-at-once，磁盘同步）下刻录且速度不超过x8。

sidux aims to be 100% compatible with Debian sid, however sidux may provide temporary hot-fixes from time to time. The sidux apt-repository holds sidux specific packages which includes the sidux kernel, scripts, tools and documentation.

Debian sid/unstable的稳定性

sid是Debian不稳定库的一个名号。Debian sid一个频繁更新的库。它与仍在维护中的软件最新最好的上游版本快速达成同步。由于频繁的更新，对软件包的整体测试可能就会因上游与已发布Debian软件包之间的时间缩短而减少。

软件包管理

sidux is compliant with Debian packaging and uses apt and dpkg for software package management with debian and other repositories identified by the files in /etc/sources.list.d/*

Debian sid有超过20,000个软件包，因此通过apt-cache搜索软件包 or with Debian Package Search GUI application.。

要安装那个软件包用apt-get install <package name>安装一个新的软件包

Debian sid's repositories can be updated as much as four times a day so running apt-get update is essential before installing any new packages or before running apt-get dist-upgrade to keep up-to-date with the server's list of packages.

Using other Debian based distributions' repositories, Source and RPMs

Source installs are not supported. If you really need to compile your application, do it as user, and put it under your home directory without installing it to the system. Using checkinstall and converting RPMs with alien, (and others like it), to make a deb, is not supported.

Other well known, (and lesser known), Debian based distributions' which repackage Debians' packages for their own repositories, often use different file locations for various applications which differ from Debian and could cause system instability and some packages won't install due to unresolvable dependencies from different package naming schemes or odd version numbers. For instance a different version of glibc could cause the application to not even run.

Use Debian repositories to install your required software packages, as other repositories will most likely not be able to be supported.

升级系统－dist-upgrade

apt-get dist-upgrade是支持的升级sidux的方案。用任何图形化的前端更新sidux都是不支持的。升级已安装的系统－dist-upgrade

仅支持在X外进行的dist-upgrade。 从你的窗口管理器（KDE，XFCE等）运行init 3或进入一个虚拟终端（ctrl＋alt＋f1，ctrl＋alt＋f2等）将停止X的运行，让你得以安全地升级。

网络设置

Ceni是一个能既快速又几近无误地设置你的网络或无线网卡网络设置工具。无线设置功能可以搜寻网络，用wep与wpa加密，用无线工具或wpa_supplicant进行设置无线。如果用的是dhcp（自动进行ip地址分配）以太网直接就可以用了，或者你可以从网络掩码到域名服务器都进行手动设置。

Ceni可通过命令Ceni或ceni运行。如果没安装，你可以通过命令apt-get install ceni安装。
