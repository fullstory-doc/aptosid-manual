ändra skärmupplösning

xrandr

Supported Card Drivers
  • xserver-xorg-video-intel (since 2.0)
  • xserver-xorg-video-nouveau ( Refer to the nouveau Feature Matrix)
  • xserver-xorg-video-ati (since 6.7.192)

First step is just to type xrandr to see if it is supported, if xrandr is not supported check the xorg version and driver used.

För att ändra upplösning på din primära skärm förutsatt att ditt kort stöder det. Exempel:

xrandr --output VGA --mode 1440x900

Dubbla skärmar och xrandr

xorg.conf is deprecated, if you use free drivers. If you have an xorg.conf stanza under /etc/X11/xorg.conf.d, because you use proprietary drivers for your graphics card, you should save it now before proceeding.

xorg.conf, if present at all, is now modular, for example, each module contains everything referring a "device" for instance, the display or a mouse.

With xrandr you can configure your primary and secondary screen without restarting X, (hotplug). Med xrandr kan du konfigurera din primära och sekundära skärm utan att starta om X. xrandr ersätter xinerama samt mergedFB. Med xrandr 1.2 aktiverad är det inte säkert att den gamla konfigurationen i xorg.conf längre fungerar.

Supported Card Drivers
  • xserver-xorg-video-intel (since 2.0)
  • xserver-xorg-video-nouveau ( Refer to the nouveau Feature Matrix)
  • xserver-xorg-video-ati (since 6.7.192)

Förberedelser för xrandr-konfigurationer i en PC med dualhead

Notera: Om du använder två skärmar hela tiden bör xorg.conf ändras så att den överensstämmer med den uppsättningen.

En bärbar dator måste konfigureras dynamiskt (till skillnad mot en PC med två skärmar) och när du startar om behöver du på nytt göra konfigurationen, såvida du inte ställer in dualhead med de parametrar du använder i xrandr, du kan kopiera dessa till ett skript och lägga i ~/.kde/Autostart/.

Bekanta dig med xrandr

Det första steget är att skriva xrandr i ett terminalfönster för att se utdatan:

VGA-0 connected 1024x768+0+0 (normal left inverted right x axis y axis) 304mm x 228mm

   832x624        74.6

Here you see that only vga, for the PC (see Appendix A for output name explanation). You see the resolutions that are supported by that screen and (which is important for dual head) the maximum screen size (here 2048x768).

Konfigurationsscenarion

Grundläggande syntax

xrandr --output <output> --rate <rate> --mode <mode> --left-of|--right-of|--above|--below|--same-as <output>

Där:

  • <output> is output name (see Appendix A)
  • <rate> is refresh rate given by xrandr output (optional)
  • <mode> is resolution given by xrandr output (optional)
ändra upplösning på den primära skärmen
xrandr --output VGA --mode 1024x768
Klon

Då många externa skärmar / projektorer inte kan köras med 1280x800 kan du istället använda t.ex 1024x768, som ett exempel:

xrandr --output VGA --mode 1024x768 --output LVDS --mode 1024x768

För att stänga av den sekundära skärmen och återgå till normal upplösning gör du enligt följande:

xrandr --output VGA --off --output LVDS --mode 1280x800
Multiple display desktop

As intel GMA <=945GM/GMS looses 3d support with a virtual screen >2048x2048, you cannot put both screens next to each other in high resolution, both at 1024x768 work well:

xrandr --output LVDS --mode 1024x768 --output VGA --mode 1024x768 --left-of LVDS

För att avaktivera "multi screen" stänger du av den sekundära skärmen och återgår till normal upplösning:

xrandr --output VGA --off (--output LVDS --mode 1280x800)

Ett annat alternativ är att placera den sekundära skärmen ovan eller under den primära:

xrandr --output LVDS --mode 1280x800 --output VGA --mode 1280x1024 --above LVDS

Resultatet är en virtuell skärmupplösning på 1280x1824 vilket är under 2048x2048. En annan lösning kan vara att rotera skärmen:

xrandr --verbose --output LVDS --mode 1280x800 --output VGA  --mode 1024x768 --rotate left --left-of LVDS

OBS: Detta fungerar bara om du även kan rotera din fysiska skärm.

Example of a permanently configured PC with dual monitors with xrandr with code snippet in /etc/X11/xorg.conf.d/30-screen.conf:
#30-screen.conf
Section "Monitor"
	Identifier      "DVI-0"
	Option          "Primary"       "true"
EndSection

Section "Monitor"
	Identifier      "DVI-1"
	Option          "RightOf"       "DVI-0"
EndSection

Section "Device"
	  Identifier     "ATI Radeon HD 2600"
	  Option         "Monitor-DVI-0" "DVI-0"
	  Option         "Monitor-DVI-1" "DVI-1"
EndSection

Notera

  • Virtuell skärmupplösning är begränas till 2048x2048 för intel. även om det är möjligt att använda en högre upplösning går du miste om DRI-stöd. Det finns dock ingen begränsning hos nvidia eller ati.
  • TV-ut fungerar inte med ATI
  • Om "DDC probing" inte fungerar korrekt med ATI (Xorg.0.log: (WW) RADEON(0): DDC2/I2C is not properly initialised) är det inte säkert att du kan överskriva värdena med modelines
  • När du fÖrsöker konfigurera ett stort skrivbord (dual head) och xrandr säger att upplösningen du efterfrågar är större än vad xrandr stöder använder du "Virtual" och den önskade upplösningen. (Se skärmdelen i Appendix A)
  • För alla grafikkort, bortsett från intel, bör den virtuella skärmupplösningen vara tillräckligt stor för båda skärmarna. Exempel: monitor1 = 1024x768 och monitor2 = 1280x1024, då bör den virtuella upplösningen vara (1024+1280)x(1024>768) -> 2304x1024
Appendix A
Intel
Output names:
* LVDS: internal laptop panel
* TMDS-1: external DVI port
* VGA: external VGA port
* TV: external TV output
ATI
Output names:
* LVDS: internal laptop panel
* DVI-0: first external DVI port
* DVI-1: second external DVI port (if present)
* VGA-0: first external VGA port
* VGA-1: second external VGA port (if present)
* S-video
Nvidia
nv driver supports RandR1.2 on G80 boards
Output names:
* LVDS: internal laptop panel
* DVI0: first external DVI port
* DVI1: second external DVI port (if present)
