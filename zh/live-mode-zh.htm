sidux-*.iso - Live模式下的root用户密码

请注意：任何时候以root用户许可权做事，你都应该知道你在做什么！在 the internet［译注：Local Area Network，局域网］范围内的网上冲浪，是无需root用户访问权的。

sidux-*.iso default password

在sidux LiveCD上，root用户密码是未设的。如果你想运行一个要求root用户特权的程序，你有几个选择：

Simple method, just type:

su
要设置一个（暂时的）root用户密码

打开一个konsole／shell：

sidux@0[sidux]$ sudo passwd
Enter new UNIX password:
Retype new UNIX password:
passwd: password updated successfully
sidux@0[sidux]$

现在你可以在后续的sidux Live对话中使用这个密码了。

sudo is not supported on hard disk installations, see sudo and sux on how to set an alias.

Running a program from a root-shell

键入su-to-root -X -c <application>将变你为带Xapp特权的root用户。

打开一个konsole／shell：

aptosid@0[aptosid]$ su-to-root -X -c gparted  （或任意你所想 ...）

Some KDE applications require dbus-launch in front of the application:

dbus-launch <Application>

Another option generic to all major Desktop Managers is:

Alt+F2 su-to-root -X -c <Application>

For example:

Alt+F2 su-to-root -X -c gparted

在konsole要是不想再用root用户，键入：

exit

或者点击右上角关掉konsole。

In case of a lockout on a sidux-*.iso you need to do the following and follow the prompts to set a password:

alt+ctrl+F1
sudo passwd

Once the password is active, action the following:

alt+ctrl+F7

Live-CD运行时安装软件

apt-get update
apt-get install 你偏好的软件包

注意：当你退出Live-CD后，所有变更都不会保存。, except if you enable fromiso and persist.

怎样通过ntfs-3g在NTFS分区执行写操作

警告：尽管ntfs-3g声称‘稳定’，千万别在没有外部备份的情况使用它，当然还不能是用来做事的系统！如果你那么做导致数据丢失，就是你的错了。所以使用请当心风险！

打开一个shell，敲入下面的命令：见对你的硬盘分区－磁盘命名

sux
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
要退出konsole，键入：exit

现在你的NTFS卷应该已挂载rw，你应该能在它上面存数据了。然而再次警告！紧急情况下才使用它，不推荐日常使用。
