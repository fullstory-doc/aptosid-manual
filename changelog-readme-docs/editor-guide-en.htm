utf-8 and dejavu

You must set all your editor defaults to utf-8 and the font to dejavu. This will enable most latin type languages to make use of all the special accents.

Naming Conventions

The xxx-xx-xx.htm is common to all languages and never to be altered, except to reflect which language it is. Therefore wm-kde-de or wm-kde-en or wm-kde-fr. You may find links in the content that points to an -en.htm, please change it when you are doing translating to reflect your language tag.

The div class="divider" id="xxxxx> is used to create the submenus and is common to all languages,, and never be altered

Warning: it passes, however let me know about it,
Red: Errors NEED to be fixed.

Kdesvn

apt-get install kdesvn
apt-get install kdesvn-kio-plugins
ADVANCE WARNING: You do not need to 'checkout' the repo again once you have it locally.:

All you need to do once you have opened kdesvn is click on File>Open and go to your local repo, click on Trunk/manual and it will automatically start to update any changed files (it will ask you for your username/password). Then click on Updates , Check if current working copy has items with newer version in the repo. The default colour for a file that was changed in svn compared with your local file will turn green. You then click update to head

The only time you will need to do another checkout is if you have corrupted your current local repo, and in that case go back and redo the 'Steps to set up your local repo'.

Steps to the manuals svn repo

  1. Make an svn folder on your own PC and inside that folder make another folder called aptosid-manual-berlios-repo or or aptosid-man-trans, (as examples)
  2. Open kdesvn
  3. In the menu: Subversion> General>Checkout a repo
  4. In the dialog box that pops up, ENTER URL put in this link to get the WHOLE repo (which is about 200 MB) 
    svn+ssh://developername@svn.berlios.de/svnroot/repos/fullstory-doc/trunk
  5. In the SELECT TARGET DIRECTORY click the folder icon and browse to your chosen folder that will have the repo locally (eg sid-man-trac-repo or whatever you have named it)
  6. Radio button HEAD
  7. Check (X) the lines: Append source URL name, Recursive, Open after job
  8. Click OK , enter your username and password, the svn repo is now incoming!

When you see a dialog box asking you to accept/fingerprint a certificate, use your mouse to stretch out the dialog box so that you see an 'Accept' button, and accept the self issued certificate.

Making changes to files
  1. Open up /kwrite/kate, browse to where you have the repo locally (eg sid-man-trac-repo) go into 'trunk' ( all the langs will come up) and go to your lang open up the file you wish to edit, edit it, save it.
  2. Open kdesvn click on File>Open and go to your local repo>'trunk". it will now have a coloured line (default is red), thus showing you that a file changed, Note if have have kdesvn open will working in a text editor , click 'reload' in kdesvn
  3. Right Click on the file showing up that it has changed and a popup menu will have options, click commit # and fill in the comments section as to what changed! , (it will/may ask your username and password if you are not already logged in )

The Blame file

To see what actually changed kdesvn has a blame file. Highlight the file you wish to see that was changed and click on the icon that says BLAME Output revision in line OR, highlighting the file and then go to the Subversion > General >Blame

NOTE: The Blame file only shows the line numbers and who changed them. To get the full explaination of the change, highlight the file and click theLOG icon, and then click on the Diff Previous button bottom left.

Basic commands:

Update your working copy:

cd /folder/to/your/working/copy
svn update

To change content of a file: just edit the file with your preferred editor then commit.

Check your changes:

svn status
Note: This will only show you the changes you made in your working directory

svn status --show-updates
Note: this will contact the repository and determine changes that were made since your last svn update eg:
svn status -u
A               0   foo/blah.txt
A               0   foo/blah2.txt
A               0   foo
M             128   xx/cd-content-xx.htm
Status against revision:    128

Note: The letters in the first column have the following meaning:
A - item is scheduled for addition
C - there is an conflict with the item
D - item is scheduled for deletion
M - item was modified

To check what content you changed use:

svn diff
Note: You'll get the changes in the unified diff format.

Undo working changes:

svn revert <path/to/file>
Note: Revert can undo any kind of scheduled operations, you'll get the version from your last svn update without the need to connect to the repository, so revert is a offline operation!

Commit your changes:

svn commit
Note: A log message must be provided, either by --file or --message, if both flags are not set a editor (vi) will be started. The log message can be empty.
How to handle conflicts:

This topic can be very complex, The official documentation: http://svnbook.red-bean.com/en/1.4/svn-book.html#svn.tour.cycle.resolve

Check logfiles:

svn log
Note: If you do a svn log (other commands act similar) in the root directory of your local working copy you get the log messages of that folder and all subfolders, if you are in a subfolder and run svn log you just get the logs of that folder and its subfolders.

Scenario for a working cycle:

svn update
svn mkdir foo
vi foo/bar.txt
add foo/bar.txt
svn diff
svn status --show-updates
(just continue if there are no conflicts (C)
svn commit --message "Just added foo/bar.txt"
