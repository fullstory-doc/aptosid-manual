Свързване към SAMBA (Windows-базирани) споделени мрежови ресурси

Всички команди в конзолата, изпълнявайте като root. Използвайте Konqueror (стартиран нормално като потребител) за навигиране в мрежовите ресурси.

server = името, или IP адреса на сървъра (компютъра с Windows на който са споделени ресурсите)
share = името на споделената директория

В KDE - Konqueror , напишете в адресното поле URL smb://server или пълния URL адрес smb://server/share

Всички ресурси (директории) споделени от конкретен сървър:

smbclient -L server

За да монтирате локално мрежова директория (достъпна за ВСИЧКИ с пълни права) - точката на монтиране (т.е. директорията в която ще монтираме мрежовия ресурс) трябва предварително да съществува. Ако не, първо я създаваме със (името на директорията е каквото искаме):

mkdir -p /mnt/server_share

Сега мрежовия ресурс може да бъде монтиран в така създадената директория - при файлова система на Windows-машината съответно VFAT:

mount -t cifs -o username=Administrator,file_mode=0777,dir_mode=0777 //server/share /mnt/server_share

,или NTFS:

mount -t cifs -o username=Administrator,file_mode=0777,dir_mode=0777,lfs //server/share /mnt/server_share

За да размонтирате ресурса:

umount /mnt/server_share

Ако искате мрежовия ресурс да е постоянно монтиран, добавете реда за него във файла /etc/fstab :

//server/share  /mnt/server_share       cifs  defaults,username=your_username,password=**********,file_mode=0777,dir_mode=0777   0   0

Споделяне на мрежови ресурси от sidux (sidux като Samba сървър)

Тъй като samba сървърът не е включен в live-cd-то на sidux, за да споделяте директории към Windows компютри трябва първо да инсталирате сървъра samba:

su
apt-get update
apt-get install samba samba-tools smbclient cifs-utils samba-common-bin

За sidux инсталиран на хард-диска:

Example 1:

Тъй като samba сървърът не се инсталира по подразбиране, за да споделяте директории към Windows компютри трябва първо да инсталирате сървъра samba:

За изчерпателна информация относно Samba и настройване Linux Samba сървър, препоръчваме документацията на Samba..

Следва прост пример за настройка на samba:

отворете файла с настройките /etc/samba/smb.conf в текстов редактор (напр. kedit или kwrite), и въведете следното (модифицирайте където е нужно, за да отговаря на вашата система):

# Промени в global секцията - всичко максимално просто,
# без пароли или защита, като в Windows 9x

[global]
security = share
workgroup = WORKGROUP

# Споделяме само с разрешение за четене - това е съществено ако споделяте NTFS-базирани файлови системи през samba!

[WINDOWS]
comment = Windows Partition
browseable = yes
writable = no
path = /media/sda1 # <--директорията която споделяте (ако трябва променете, за да отговаря на вашия раздел)
public = yes

# Споделяме с разрешение и за четене и за запис - раздела трябва да бъде монтиран
# разрешение за запис - в този пример за споделяне на FAT32 директория през samba, е безопасно

[DATA]
comment = Data Partition (first Extended partition)
browseable = yes
writable = yes
path = /media/sda5
public = yes

Restart the samba server

/etc/init.d/samba restart

Example 2:

groupadd smbuser
useradd -g smbuser <the-user-you-want>
smbpasswd -a <the-user-you-want>
smbpasswd -e <the-user-you-want>

Next edit /etc/samba/smb.conf to give it share permissions, (be careful with what folders you enable), for example:

[homes]
   comment = Home Directories
   browseable = yes.
   writeable = yes

[media, be careful!]
    path = /media
    browseable = yes
    read only = no
    #read only = yes
    guest ok = no
    writeable = yes

[video]
    path = /var/lib/video
    browseable = yes
    #read only = no
    read only = yes
    guest ok = no

#any other folder you want to share with windows/linux/mac
    #path = path = /media/xxxx/xxxx
    #browseable = yes
    #read only = no
    #read only = yes
    #guest ok = no

Restart the samba server

/etc/init.d/samba restart

Проверка на споделените чрез samba локални ресурси

За да дадете достъп, без оглед на сигурността (напр. за споделяне в LAN мрежа), :

Проверете правата за достъп до директорията и файловете в нея; - трябва да са минимум -rwxr-xr-x:

ls -la pathTo/dirShareName/*

Ако не са, добавете правата чрез:

chmod -R 755 pathTo/dirShareName

Ако искате също и право на запис:

chmod -R 777 dirShareName

За да се провери работи ли споделената директория: (не забравяйте първо да стартирате сървъра):

smbclient -L localhost

Горната команда трябва да изведе нещо подобно на:

 smbclient -L localhost
Password:
Domain=[HOME] OS=[Unix] Server=[Samba 3.0.26a]

        Sharename       Type      Comment
        ---------       ----      -------
        IPC$            IPC       IPC Service (3.0.26a)
        MaShare         Disk      comment
        print$          Disk      Printer Drivers
Domain=[MSHOME] OS=[Unix] Server=[Samba 3.0.26a]

Ако сте дали достъп без парола, просто натиснете ENTER

След промените във файла с настройките на samba, запишете ги на диска. Стартирането (или спирането) на samba, става съответно чрез командите:

/etc/init.d/samba start

и

/etc/init.d/samba stop

Ако искате да включите демона на samba, да се стартира автоматично при рестартиране на компютъра:

update-rc.d samba defaults

Сега samba сървъра автоматично ще се включва при стартиране на компютъра, и ще спира при изключване.
