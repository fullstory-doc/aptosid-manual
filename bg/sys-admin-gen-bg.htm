Включване и изключване на услугите (фоновите програми, демоните) в sidux

insserv (от версия Pontos 2008-04 нататък): За да включите/изключите вече инсталирана услуга:

Прочетете подробно /usr/share/doc/insserv/README.Debian, забележките и man-докуметацията:

 $ man insserv
 $ man invoke-rc.d
 $ man update-rc.d
 търсете в google за LSB headers

За стартиране, 'start':

service <service> start

За спиране, 'stop':

service <service> stop

За рестартиране, 'restart:

service <service> restart

За да изключите услуга за постоянно, т.е. да не се включва при зареждането на системата

update-rc.d <service> remove
[ще премахне всички стартиращи симлинкове за съответните нива на работа]

За да се уверите, че услугата ще се включва при зареждането на системата [не винаги е необходимо]

update-rc.d <service> defaults
[ще направи стартиращите симлинкове в съответните нива на работа]

To read the list of what services are running:

ls /etc/rc5.d

S means the service will start.
K means service will not start.

По-стари инсталации (преди Pontos 2008-04)

insserv на инсталирани системи преди Pontos 2008-04

Предупреждение: По-старите инсталации при обновяване няма да получат insserv автоматично, тъй като евентуално може да имат собствени стартиращи скриптове или симлинкове които да влязат в конфликт с insserv. НЕ СЕ препоръчва инсталирането на insserv на системи преди Pontos. Пакетът трябва да бъде използван внимателно - тъй като неправилни или отсъстващи LSB-зависимости в съществуващите стартиращи скриптове, могат да доведат до система отказваща да се стартира. Инсталацията на insserv за съществуваща инсталация, може да се нуждае от някои ръчни промени.

Прочетете документацията на Debian, и забележките към пакета, ппеди да инсталирате и включите insserv.

Boot-Up Manager (bum) - Graphical services configuration tool

If the logic of a debian system boot up sequence is not very clear and familiar to you, you should not play with symlinks, permissions and so forth. In order to avoid messing up your system. Boot-Up Manager will help you to automate your configuration.

Boot-Up Manager is a GUI runlevel configuration editor which allows you to configure what init services are invoked when the system boots up or reboots. It displays a list of every service which could be started at boot. You can toggle individual services on and off.

apt-get install bum

To use the Boot-Up Manager GUI:

$ su-to-root -X -c bum

Detailed documentation on Boot-Up Manager (bum).

Убиване (аварийно завършване) на услуга или процес

pkill е удобна команда - тъй като работи както под root, така и под обикновен потребител. Чрез нея може да откривате процеси, и да им изпратите сигнал по име или други атрибути

pkill -n service

Ако не сте сигурни за името на процеса (или услугата) които искате да "убиете", pkill <tab> <tab> ще изведе списък с възможностите.

Също добра алтернатива е htop. (killall -9 използвайте само в краен случай).

sidux нива на работа (runlevels) - init

Това е списък с функционалността на нивата на работа (runlevels) за операционната система sidux. Имайте предвид че те се отличават от нивата на работа на debian stable:

Runlevel sidux Debian
init 0 Powers off the PC. Powers off the PC.
init 1 Single user (safety or rescue mode). Dæmons like apache and sshd are stopped. Do not go to this level via remote access. Single user, stops services (safety or rescue mode). Do not go to this level via remote access.
init 2 Multi-User mode with the network running, with the X-Window System not running, and/or to stop or not enter the X-Window System. Debians' default runlevel for Multi-User mode with the network running the X-windows system.
init 3 Multi-User mode with the network running, with the X-Window System not running, and/or to stop or not enter the X-Window System. This is where a dist-upgrade is actioned. Same as runlevel 2 / init 2.
init 4 Multi-User mode with the network running, with the X-Window System not running, and/or to stop or not enter the X-Window System. Same as runlevel 2 / init 2.
init 5 The sidux default for Multi-User mode with the network running the the X-Window System, and/or to start the X-Window System. Same as runlevel 2 / init 2.
init 6 Restart/reboot the system. Restart/reboot the system.
init S This is where early boot time services are executed on a 'once only basis'. You cannot switch to it after it has been run. This is where early boot time services are executed on a 'once only basis'. You cannot switch to it after it has been run.

To acertain the runlevel (init) you are currently in:

who -r

Required reading for any sidux and Debian system administrator regarding runlevels:

man init

Забравена root парола

Забравена парола не може да се възстанови - но може да се зададе нова.

Първо, стартирайте от Live-CD-то.

Като root, монтирайте вашия основен раздел (root partition) (например /dev/sdb2)

mount /dev/sdb2 /media/sdb2

Сега chroot-нете в стария ви основен раздел със (chroot = changed root, сменяне на основния раздел на системата "в движение") и задайте нова парола.

chroot /media/sdb2 passwd

Setting new passwords

To change your 'user' password, as $ user:

$ passwd

To change your 'root' password, as # root:

passwd

To change a user's password as an administrator, as # root:

passwd <user>

Шрифтове и размери по подразбиране

sidux използва свободните шрифтове (от Debian) които са добре балансирани; ваши собствени шрифтове могат евентуално да имат по-лошо качество на визуализация. Някои опции в Debian (apart from KDE>systemsettings ) могат да помогнат за визуализирането също и на външни шрифтове. Но имайте предвид, че всеки шрифт има само няколко наистина перфектни размера; останалите размери могат да не работят толкова добре.

Промяната на dpi също може да е от полза, поиграйте си с различни стойности:

fix-dpi-kdm

Би трябвало да покаже и зададе DPI подходящо за размерите на екрана ви, но пробвайте и с други стойности. Трябва да превключите в init 3 и после обратно в init 5 (или да рестартирате) за да видите резултата от промяната.

След настройката на шрифтовете и DPI (в X или Firefox/Iceweasel), може да донастроите в зависимост от предпочитанията, особено ако сте сменили от Bitmap на True Type шрифтове или обратно:

dpkg-reconfigure fontconfig-config

Изберете native, и сложете autohinter на automatic. Или пробвайте другите варианти.

Ако е нужно да възстановите кеша на шрифтовете:

fc-cache -f -vv

Ако това не работи, може да трябва да преинсталирате пакета с настройки по подразбиране за кеша на шрифтовете:

apt-get install --reinstall --yes -o DPkg::Options::=--force-confmiss -o DPkg::Options::=--force-confnew fontconfig fontconfig-config
GTK-базирани приложения като Firefox/Icewasel

CUPS (принтери)

KDE има голям раздел посветен на това в KDE help, но обновяването на системата често може да дестабилизира cups. Ако това се случи, едно възможно решение:

modprobe lp
echo lp >> /etc/modules
apt-get remove --purge cupsys cups
apt-get install cups
        ИЛИ
apt-get install cups cups-driver-gutenprint hplip

Уверете се че CUPS услугата е включена:

service cups restart

После в браузера: 

http://localhost:631

Друго недоразумение може да възникне когато настройвате принтерите през графичния интерфейс (в диалога който ви пита за парола и е със вашето потребителско име предварително попълнено) - ако въведете паролата си няма да работи - всъщност трябва да смените потребителското име от вашето на root, и съответно да въведете паролата за root вместо вашата.

The OpenPrinting database contains a wealth of information about specific printers, along with extensive driver information, the drivers themselves, basic specifications, and an associated set of configuration tools.

Sound in sidux

Sound is muted by default in sidux.

The KDE version uses Kmix and XFCE uses Mixer.

Often it is only a matter of clicking on the sound icon in the taskbar and unchecking the 'Mute box'.

Kmix

In Kmix you will need to activate the options you prefer for channel options, Kmix>Setting>Configure Channels. Or in a terminal:

$ kmix
XFCE

In XFCE run the mixer application and add some controls via the Multimedia>Mixer and click the Select Controls box. Or in a terminal:

$ xfce4-mixer

Alsamixer

If you prefer to use Alsamixer, it is in the alsa-utils package:

apt-get update
apt-get install alsa-utils
exit

Set your preferred sound settings as $user from the terminal:

$ alsamixer
