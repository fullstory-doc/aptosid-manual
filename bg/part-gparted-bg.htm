Основи

Физическият диск може да бъде разделен на един или повече раздели (partitions). Всеки раздел (partition) трябва да има определена файлова система. Linux може да използва различни файлови системи. Ext4 е препоръчваната файлова система за sidux. ext2 е удобна за файлова система за съхраняване на данни, тъй като съществува драйвер за MS Windows™ чрез който можем да получим достъп до нея под Windows (и например, да споделяме файлове между Linux и Windows ако имаме дуал-бут система). Ext2 Installable File System For MS Windows.

За нормална употреба препоръчваме файловата система ext4, тя е файловата система по подразбиране за sidux, отлично поддържана и надеждна.

Using KDE Partition Manager & Gparted

Creating and managing partitions is not something that is typically done every day. Therefore, a good idea is to read this guide once, to get comfortable with the concepts and some of the panels that will appear. The following assumes that you are using a live-iso . Should you be hard disk installed <sudo> will not work, refer to About Xapps whilst in X as root .

KDE Partition Manager, if installed on the live-iso - run in a terminal:

sudo partitionmanager

Gparted, if installed on the live-iso - run in a terminal:

sudo gparted
Gparted will you let create and see GPT disks.

If you need to partition a USB drive, (or SD and SDXC cards), whilst hard disk installed you will need to be root. Refer to About Xapps whilst in X as root else use the commandline tools cfdisk or gdisk.

 

  • When GParted or KDE Partition Manager runs , a window is opened, and drives are scanned.

    In KDE Partition Manager the drives are shown in a list on the left.

    In Gparted, if you click the menu (at the top left), a pop down is presented. You can select to refresh the display of the drives on your system.

The following screenshots are of Gparted. KDE Partition Manager behaves in much the same manner.

  • Редакция (Edit)

    Менюто за редакция (Edit) има три жизненоважни функции:
    Undo last operations (отменя измененията в списъка)
    Apply (прилага измененията в списъка)
    Clear all operations (изчиства всички изменения в списъка).

  • Вид (View)

    • Информация за устройството (Device Information)

      Този панел показва информацията за хард-диск устройството като Модел, Размер и т.н. Той е удобен в система с много дискове, където тя ще помогне да идентифицираме конкретния диск - и да се уверим, че искаме да променяме именно него.

    • Чакащи операции (Pending Operations)

      В долния прозорец е списъка на чакащите операции (изменения) - такива които са избрани да бъдат извършени, но все още не са приложени окончателно, физически върху диска.

  • Устройство (Device)

    Оттук за всяко устройство може да бъде определено "име" (Disk Label) или да се смени вече съществуващото, ако не е подходящо.

  • Раздели (Partition Menu)

    Частта за раздели в менюто е една от най-важните. Оттук се извършват много неща, някои от които могат да бъдат опасни.

    Изтриване (Delete) се избира ако искате да изтриете раздел от диска. За да изтриете раздела, първо го изберете с мишката.

    Промяна на размера/Преместване (Resize/Move) се използва за преоразмеряване или отместване на разделите (когато това е възможно).

  • Създаване на нов раздел

    В панела с инструментите, бутона New се използва за създаване на нов раздел, ако вече сте избрали свободното (незаето от раздели) пространство от диска (unallocated area). Ще се появи нов прозорец където трябва да укажете желания размер на раздела, типа му (Primary, Extended или Logical), името му (disk label), и желаната файлова система за него.

  • Ако направите грешка

    Ако сте направили грешка, може чрез бутона за Изтриване (Delete) да изтриете раздела; или ако още не сте приложили промените чрез натискане на бутона Apply all Operations, използвайте Undo Last Operations - бутона за отмяна.

  • Преоразмеряване/Преместване (Resizing/Move)

    Ако искате да преоразмерите (свиете, разширите) или преместите избрания раздел, използвайте Resize/Move бутона: ще се появи нов прозорец. Използвайте мишката или стрелките за промяна на размера на раздела.

    След използването на командата Resize за преоразмеряване, и когато сте се уверили че всичко е правилно, натиснете Apply, за да приложите вече физически промените върху диска.

    Времето което заема операцията зависи от размера на раздела който се обработва.

    След завършване на манипулациите с разделянето на диска, задължително трябва да рестартирате системата - за да може sidux да прочете новата таблица с раздели.

Преоразмеряване на NTFS раздели

Промяната на размера на NTFS раздел изисква незабавно след прилагането му да рестартирате системата! НЕ ИЗВЪРШВАЙТЕ никакви действия повече върху този раздел преди да сте рестартирали - ако го направите, ще получите грешки.

Open a shell and enter the following commands:See Partitioning your HD - Disk Naming

su
apt-get update && apt-get install ntfs-3g
umount /media/xdxx
mount -t ntfs-3g /dev/disk/by-uuid/xxyyzz[etc] /media/xdxx
To get out of the konsole type: exit

An example to add ntfs to /etc/fstab after installing ntfg-3g for a permanent mount point is :

 UUID=486E97C46E97A8E8                         /media/disk3part1    ntfs         defaults  0    0

Now your NTFS Volume should be mounted rw and you should be able to store data on it. But again, be warned! Use it in emergency situations, it is not recommended for use on a daily basis.

exfat

exfat is a proprietary file system covered by patents held by Microsoft ™ and commonly found on SDXC cards. To mount/read/write to the file system, simply install the exfat-fuse package:

apt-get update && apt-get install exfat-fuse
